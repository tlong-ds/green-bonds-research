ROOT CAUSE ANALYSIS
================================================================================

PRIMARY ISSUE: Identifier Type Mismatch
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

What We Have:
  - DealPermId: Internal Refinitiv deal transaction ID
  - IssuerPermId: Internal Refinitiv company/entity ID
  
What LSEG API Expects:
  - RIC: Reuters Instrument Code (for securities)
  - ISIN: International Securities Identification Number
  - CUSIP: Committee on Uniform Security Identification (US/Canada)
  - Other standard market identifiers

Why It Fails:
  The LSEG Desktop API's get_data() function validates identifiers against
  its symbol database. DealPermId and IssuerPermId are NOT in this database.
  
  DealPermId specifically:
  - Is an internal Refinitiv transaction/deal identifier
  - Does not map to tradeable securities
  - Cannot be used to query market data
  - Works in Refinitiv's web interface but not in programmatic APIs

SECONDARY ISSUE: Field Name Case Sensitivity
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Batches 4-5 produced different errors than Batches 1-3:
  "Unable to resolve all requested fields in ['TR.FIFILINGDATE', ...]"
  
  This suggests that:
  1. Field names are case-sensitive
  2. LSEG normalizes to uppercase internally
  3. Some fields may not exist in the LSEG database
  4. Or the API has stricter validation when identifiers are invalid
